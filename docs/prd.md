# My 0to1 — PRD

**버전**: 0.1 | **작성일**: 2026-04-21 | **상태**: Draft

## Problem Statement

1인/초기 창업자는 **"문제/고객부터 찾아야 한다**"는 것을 알면서도, 실제로는 솔루션 아이디어에 끌려다니는 함정에 반복적으로 빠진다. **Self map(나 자신의 이해), Problem universe(시장 문제 지도), Founder-problem fit(궁합 판단), Validation backlog(검증 실행)를 한 시스템 안에서 다루는 도구**가 없어, 결국 노션 조각·북마크·기억에 의존하다 "무엇을 할지" 고민이 반복된다. 이 문제를 해결하지 않으면 **창업자는 수개월을 아이디어 탐색에 소진하고, 정작 실제 검증으로 넘어가지 못한다.**

---

## Goals

1. **집중 가속**: 첫 사용 후 1주 이내에 "깊게 들어가 볼 문제 1~3개"를 식별한다 (Fit Judge 추천 → 사용자 셀프 평가 완료 기준).
2. **실행 전환율**: Validation backlog에 등록된 문제 중 50% 이상이 실제 인터뷰/실험으로 이어진다 (30일 기준).
3. **Self map 완성도**: 온보딩 완료 사용자의 80%가 Self map 핵심 항목(관심사·강점·혐오·몰입 경험·접근 네트워크) 5개 이상을 채운다.
4. **재방문율**: 주간 활성 사용자(WAU) 기준 40% 이상이 매주 1회 이상 에이전트와 대화한다.
5. **납득 경험**: 사용 4주 후 설문에서 "이 문제를 정말 풀어도 되겠다는 확신이 생겼다" 응답률 60% 이상.

## Non-Goals (v1 범위 밖)

| 항목 | 이유 |
| --- | --- |
| 팀 협업 기능 (멀티 유저) | v1은 철저히 1인 창업자 개인 도구. 팀 기능은 페르소나와 UX가 근본적으로 다름 |
| 투자자 매칭 / 네트워킹 | Discovery 단계 이후의 영역. 현재 핵심 루프와 인접하지만 별도 제품 |

## User Stories

### Persona A — 솔로 창업 준비자 (아직 문제를 못 잡은 단계)

- As a 솔로 창업 준비자, I want **Self Insight Agent가 나에게 질문을 던져 대화하듯 내 관심사와 강점을 끌어내도록 해서, 내 Self map을 채우는 데 드는 인지 부하를 줄이고 싶다.**
- As a 솔로 창업 준비자, I want **나와 잠재적 fit이 높아 보이는 문제 카드 목록을 추천받아서, 무한한 아이디어 목록이 아니라 내게 의미 있는 후보를 빠르게 탐색하고 싶다.**
- As a 솔로 창업 준비자, I want **각 문제에 대해 끌림·이해도·접근성·장기 동기를 직접 점수화하고 그 결과를 볼 수 있어서, "이 문제가 나와 맞는가"를 감이 아닌 구조로 판단하고 싶다.**

### Persona B — 현직 PM / 커리어 전환 고민자

- As a 현직 PM, I want 실제 Seed~Series A 스타트업이 풀고 있는 문제 카드를 시장 증거와 함께 탐색할 수 있어서, "이미 시장에서 의미 있게 다뤄지는 문제"에서 내 출발점을 찾고 싶다.
- As a 현직 PM, I want Reality Check Agent가 "이건 솔루션이 먼저인 것 같다"고 편향을 지적해 줘서, **내가 모르는 사이에 빠지는 솔루션 중심 사고를 스스로 알아차리고 싶다.**

### Persona C — 이미 문제 후보를 가진 1인 창업자

- As a 1인 창업자, I want 선택한 문제와 솔루션 가설에 대해 **4가지 핵심 가설(문제 존재 / 심각도 / 솔루션 핏 / 지불 의사)별로 가장 싸고 빠르게 깨는 검증 메서드와 성공·실패 시그널을 처방받아서**, 어떤 한 걸음을 다음에 둘지 고민 없이 바로 실행으로 넘어가고 싶다.
- As a 1인 창업자, I want **막막할 때 솔루션 후보 3개를 에이전트로부터 받아 그중 하나를 편집·확정해 검증 가설로 등록**할 수 있어서, 백지에서 솔루션 가설을 짜내는 부담을 덜고 싶다.
- As a 1인 창업자, I want **활성 솔루션 가설이 broken으로 결정되면 같은 문제 위에서 새 솔루션 가설을 추가해 다시 검증을 시작**할 수 있어서, 문제 단위로 누적된 검증 결과(존재 여부·심각도)는 그대로 두고 솔루션만 갈아끼울 수 있게 하고 싶다.
- As a 1인 창업자, I want **인터뷰·실험에서 배운 점을 기록하면 그 내용이 Self map과 Problem universe에 자동 반영되어서, 매번 흩어진 메모를 찾아 헤매지 않고 누적된 인사이트 위에서 다음 루프를 시작하고 싶다.**

## Requirements

### Must-Have (P0) — 이게 없으면 핵심 문제를 풀지 못한다

**Self Insight Agent — 대화형 Self map 구축**

- 에이전트가 주도적으로 질문을 던지는 인터뷰 형식 UX (일방향 폼 아님)
- 답변을 구조화해 Self map DB(관심사·강점·혐오·몰입 경험·네트워크)에 자동 저장
- 사용자가 Self map 현황을 언제든 확인하고 수정 가능
- *Acceptance criteria*: 첫 세션 20분 이내에 Self map 핵심 항목 3개 이상 채워짐

**Problem Scout Agent — 문제 카드 수집 및 표시**

- 문제 카드 구성: 누가 / 언제 / 왜 겪는지 + 현재 대체재 + 불편 요소
- 국내외 Seed~Series A 스타트업 기반 최소 100개 문제 카드 (런치 시점)
- 카테고리/키워드 필터링 가능
- *Acceptance criteria*: 사용자가 검색어 없이 탐색해도 관련 문제 카드를 20초 내 발견 가능

**Fit Judge Agent — 문제 추천 + 셀프 평가**

- **Self map 기반으로 fit 가능성 높은 문제 카드 추천 (상위 5~10개)**
- **각 문제에 대한 셀프 평가 UI: 끌림·이해도·접근성·장기 동기 (4개 축, 1~5점)**
- **평가 결과를 기반으로 우선순위 1~3개 문제 정리 뷰**
- *Acceptance criteria*: 평가 완료 후 "내 상위 문제" 뷰가 자동 생성됨

**Validation Designer Agent — 4가설 검증 처방**

- 가설(문제 존재 / 심각도 / 솔루션 핏 / 지불 의사) **axis별로 처방** 산출:
    - 추천 검증 메서드 (인터뷰·관찰·스모크 테스트·페이크 도어·소액 선결제·컨시어지 PoC 중)
    - 성공/실패 시그널
- 솔루션 가설 generation은 책임 밖 — Solution Suggester가 담당.
- 데이터 단위: existence/severity는 ProblemCard에, fit/willingness는 SolutionHypothesis에 매달리는 `Hypothesis` 레코드.
- *Acceptance criteria*: 활성 솔루션의 4가설 카드를 보고 사용자가 "다음 검증 한 걸음(어떤 메서드로 누구에게)을" 즉시 특정할 수 있음.

**Solution Suggester Agent — 솔루션 후보 생성**

- 사용자가 막막할 때 호출해 **솔루션 statement 후보 3개**를 산출.
- 입력 컨텍스트: 문제 카드 + Self map 핵심 + 문제 검증 findings + 기존 SolutionHypothesis 목록(중복 회피).
- 출력은 statement 텍스트만 — 사용자가 그중 하나를 골라 편집·저장해야 SolutionHypothesis로 등록됨 ("사용자를 대신해 결정하지 않는다" 원칙).
- *Acceptance criteria*: 사용자가 "후보 받기 → 1개 선택·편집 → 저장"을 1분 이내에 끝낼 수 있음.

**OnePager Composer Agent — 솔루션 1-pager 초안 작성**

- 4축 검증(존재·심각도·핏·지불 의사)이 커버하는 **Desirability** 외에 **Feasibility**(MVP 만들 수 있나)와 **Viability**(만들어도 굴러가나)를 강제로 사고하게 만드는 그릇. 1-pager 자체는 검증 가설이 아니라, 사업화 사고를 트리거하고 Reality Check의 입력을 풍부하게 하는 *scaffold*.
- 활성 솔루션 1개에 대해 10 섹션 1-pager 초안을 한 번에 생성: **한줄 요약 / 타깃 고객 / 문제 / 솔루션 / MVP 범위(min set of features) / MVP 구현 비용 추정 / 운영 모델 / 수익화 가설 / 주요 리스크 3개 / 30일 이내 검증 액션**.
- 입력: ProblemCard 풀 + SolutionHypothesis statement + 자식 4축 Hypothesis(status·findings 200자 요약). Self map은 주입하지 않음(토큰 폭발 방지).
- 출력: 10개 섹션 한국어 텍스트, JSON mode + zod.
- 트리거: 진입 조건(존재·심각도 둘 다 confirmed + 솔루션 active) 만족 시 `SolutionValidationBlock` 안에 "1-pager 초안 생성" CTA 노출. 진입은 사용자 결정 — 자동 생성 안 함.
- 책임 경계: 초안 생성만. 의견·추천·결정 표현 금지. 사용자가 모든 섹션 자유 편집 — Solution Suggester 패턴과 동일하게 "AI 후보 → 사용자 편집"으로 자기결정 원칙 준수.
- 데이터 단위: SolutionHypothesis 1:1 unique. 한 솔루션 = 1-pager 1개.
- *Acceptance criteria*: CTA 클릭 후 10초 내에 초안 10 섹션을 받고, 즉시 편집 가능한 상태로 저장됨.

**✅ 루프 연결 — 학습의 Self map 반영**

- 인터뷰/실험 결과를 기록하면 Self map·Problem universe·Fit 평가에 반영
- *현재 상태*: 가설별 `findings` 텍스트 필드로 1차 적재. 정식 `LearningLog` 엔티티 + 역류 규칙은 Phase 2에서 도입 예정.
- *Acceptance criteria*: 기록 후 관련 Self map 항목 또는 문제 카드가 업데이트됨

**Reality Check Agent — 편향 지적 + 증거 요구**

- "솔루션이 먼저인 것 같다", "이건 희망사항일 수 있다" 등 편향 지적
- 더 필요한 증거/질문 제안
- **컨텍스트**: 1-pager가 있으면 10 섹션 풀 컨텍스트(MVP 비용·운영·수익화·리스크 포함)를 보고 페르소나 4명이 비판. 없으면 솔루션 statement + 4축 검증 상태만으로 동작(하위호환). RC FK는 SolutionHypothesis 그대로 — 입력만 풍부해짐, 스키마 마이그레이션 없음.
- *현재 상태*: 구현 완료 (냉정한 투자자 / 솔직한 친구 / 소크라테스식 질문자 + 모더레이터 4 페르소나). 1-pager 통합은 OnePager Composer 도입 후 진행.

### Nice-to-Have (P1) — 있으면 경험이 크게 좋아진다

**Problem Scout 주기적 업데이트**

- 신규 스타트업/투자 뉴스 기반으로 문제 카드 자동 갱신 (주 1회)
- 사용자에게 "이런 새 문제 카드가 추가됐어요" 알림

**Weekly Digest**

- 주간 Self map 변화 요약 + 이번 주 추천 문제 + Validation backlog 진행 상황

**문제 카드 직접 추가**

- 사용자가 외부에서 발견한 문제를 직접 카드로 등록
